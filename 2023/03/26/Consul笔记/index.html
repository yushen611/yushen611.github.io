consul笔记

Posted on 2023-03-26 In go
consul使用笔记。当前进度：consul的概念。

I. 前言

A. 前置知识

在学习Consul之前，建议掌握以下技术和概念：

  1. 网络基础知识：包括IP地址、端口、网络协议、TCP/IP、HTTP等基础网络概念和技术。
  2. 服务架构和微服务架构：了解服务的定义和服务架构的基本原理，理解微服务架构的概念和特点。
  3. Linux操作系统基础：熟悉基本的Linux命令行操作和系统管理。
  4. Docker容器：了解Docker容器的概念、使用方法和基本原理。
  5. DevOps文化：理解DevOps（Development and Operations）文化的理念、原则和实践方法。

虽然了解以上知识可以帮助更好地理解Consul，但并非必须的。Consul提供了易于使用的API和CLI工具，使得初学者也能够快速上手使用。

DevOps（Development and Operations）是一种软件开发和运维的文化和工作方式，强调软件开发和IT运维之间的协作和沟通，旨在通过自动化和持续交付等实践，加速软件开发和部署的速度和质量，提高软件交付的效率和可靠性。

DevOps的核心理念包括持续集成、持续交付、持续部署、基础设施即代码、自动化测试、容器化等。这些实践方法可以帮助团队在开发和部署软件时更加快速、高效、可靠地交付高质量的软件，同时也能够更快地响应市场需求和客户反馈，提高业务价值和竞争力。

在DevOps文化中，开发团队和运维团队不再是孤立的，而是密切协作和共同负责整个软件生命周期。开发人员编写代码和测试代码的同时，运维人员负责部署和管理基础设施，并提供自动化工具和流程支持。这样，整个团队能够更快地响应市场变化和客户需求，更快地交付软件，并不断优化和改进软件质量和性能。

B. Consul 简介

Consul是一种开源的服务发现、配置管理、安全认证的平台，由HashiCorp公司开发。它是一个分布式系统，可以用于服务发现和注册、健康检查、故障恢复、配置管理、安全认证等方面。

Consul的核心功能包括：

  1. 服务发现和注册：Consul可以让服务在集群中自动发现和注册，同时提供了多种服务发现方式，包括DNS、HTTP API和gRPC等。
  2. 健康检查：Consul可以对服务进行周期性的健康检查，如果服务异常，则会从服务发现列表中将其移除，防止客户端调用不可用的服务。
  3. 故障恢复：Consul可以自动监控服务健康状况，并进行故障转移和恢复操作。
  4. 配置管理：Consul可以将应用程序的配置信息集中存储，并提供API进行访问和修改，支持动态更新配置。
  5. 安全认证：Consul提供了一套完整的安全认证机制，包括访问控制列表（ACL）、TLS加密等，可以保证集群中服务的安全性和可信度。

Consul的优点包括易于使用、高可用、扩展性强、支持多种语言和平台、提供了丰富的API和UI等。它广泛应用于微服务架构、容器编排、配置管理、安全认证等领域。

C. Consul 的应用场景

  1. 服务发现和负载均衡：Consul 提供了服务注册和发现功能，能够自动检测服务的健康状态并实现流量的负载均衡。
  2. 分布式配置管理：Consul 提供了 K/V 存储功能，可以存储和管理分布式系统的配置信息，能够实时更新配置并同步到各个节点。
  3. 健康检查和故障恢复：Consul 提供了丰富的健康检查机制，能够对服务进行主动和被动的检查，并通过自动化故障恢复机制实现服务的高可用性。
  4. 安全认证和授权：Consul 提供了 ACL（Access Control List）机制，能够实现对服务和配置的安全认证和授权。
  5. 可观测性和监控：Consul 提供了丰富的监控和可观测性功能，能够监控服务的健康状态、流量和错误率，并提供了多种方式进行监控和告警。

A. 安装 Consul

B. Consul 基础概念

1.服务 2.节点 3.健康检查 4.K/V 存储 5.其他

  1. 服务（Service）：一个服务是一个可执行的软件进程，可以是 Web 服务器、数据库、消息队列等。
  2. 节点（Node）：节点是运行服务实例的主机。
  3. 健康检查（Health Check）：Consul Agent 可以定期检查每个服务实例的健康状况，通过发送一个 HTTP 请求或 TCP 连接来检查服务是否可用。
  4. K/V 存储（K/V Store）：Consul 提供了一个轻量级的 K/V 存储，可以用来存储配置信息或其他任意数据。
  5. 服务注册（Service Registration）：Consul 允许服务将自己注册到集群中，以便其他服务可以发现并使用它。
  6. 服务发现（Service Discovery）：Consul 允许服务发现其他服务的位置，并使用它们提供的功能。
  7. Consul Web UI：Consul 提供了一个 Web 界面，可以用于监视和管理 Consul 集群中的服务和节点。

C. Consul Web UI

1.Web UI 概览

Consul Web UI 是 Consul 自带的一个 Web 界面，提供了丰富的功能，可以帮助用户监控和管理 Consul 集群中的服务和节点。

一些常用的功能包括：

  1. Services：列出了集群中所有的服务，并显示它们的状态和标签。
  2. Nodes：列出了集群中所有的节点，并显示它们的状态和标签。
  3. Health Checks：显示了所有服务实例的健康检查状态，可以让用户快速了解哪些服务实例正常，哪些出现了问题。
  4. Key/Value：提供了一个简单的 K/V 存储接口，可以让用户在 Consul 中存储和检索数据。
  5. Configuration：提供了一些用于配置 Consul 的选项，可以让用户对集群的行为进行微调。
  6. Consul members：显示了所有的集群成员，包括服务器和客户端。
  7. ACLs：用于管理访问控制列表（ACLs），以限制对 Consul 集群的访问。

2.使用 Web UI 操作 Consul

  1. 注册服务：在 Web UI 中，选择“Services”选项卡，点击“Register Service”按钮，在弹出的对话框中填写相关信息即可完成服务的注册。
  2. 发现服务：在 Web UI 中，选择“Services”选项卡，可以查看所有已注册的服务，并查看每个服务的实例。可以通过检查服务的健康检查状态来确定实例是否可用。
  3. 查看节点：在 Web UI 中，选择“Nodes”选项卡，可以查看所有已注册的节点，并查看每个节点的健康状况。
  4. 查看健康检查状态：在 Web UI 中，选择“Health Checks”选项卡，可以查看所有服务实例的健康检查状态，可以通过筛选器来查看特定服务实例的健康检查状态。
  5. 存储和检索数据：在 Web UI 中，选择“Key/Value”选项卡，可以存储和检索数据。可以创建键值对，并将其存储在 Consul 中。还可以使用 Consul 的 HTTP API 从应用程序中访问这些数据。
  6. 监控 Consul 集群：在 Web UI 中，选择“Consul members”选项卡，可以查看所有的集群成员，包括服务器和客户端。可以查看成员的健康状况，以及它们之间的通信状态。

D.API和CLI工具

Consul提供了易于使用的API和CLI工具，使得初学者也能够快速上手使用。

工具 命令 说明
API curl -X PUT -d '{"name": "web", "address": "10.0.1.5", "port": 80}' http://localhost:8500/v1/agent/service/register 注册一个名为”web”的服务
API curl -X GET http://localhost:8500/v1/health/service/web?passing=true 获取所有名为”web”的健康服务
API curl -X PUT -d '{"datacenter": "dc1", "node": "node1", "checks": [{"name": "HTTP Check", "http": "http://localhost:8080/health", "interval": "10s"}]}' http://localhost:8500/v1/agent/service/register 注册名为”HTTP Check”的健康检查
CLI consul agent -dev 启动Consul Agent
CLI consul join <address> 将当前Agent加入到另一个Agent所在的集群中
CLI consul members 查看当前集群的成员列表
CLI consul catalog services 列出当前集群中注册的所有服务
CLI consul catalog nodes
